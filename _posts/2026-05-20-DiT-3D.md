### 세 줄 요약
DiT 구조를 3D pointcloud 데이터인 Shapnet에 적용하고자 하였다. 가장 핵심적인 부분은 2D의 DiT를 3D에 어떻게 접목시킬지와 그로 인해 파생되는 문제점들(지나치게 큰 학습 시간과 메모리 사용량)을 어떻게 효과적으로 해결하는지이다. 논문 자체는 여러가지 삥땅을 친 흔적이 보이지만, 2D에서 DiT구조가 구조적인 큰 해결책을 제시한 것과 마찬가지로 본 논문 역시 우후죽순으로 피어나던 pointcloud 생성 모델에서 하나의 효과적인 단일 구조를 제시했음이 의미가 있다. 


### 1. Introduction
이미지 도메인에서는 DiT구조가 지배적인 입지를 다졌지만 3D shape generation에는 아직 DiT 구조가 적용되지 않았다. 현존하는 모델들은 U-Net backbone을 이용중이다.

3D shape generation을 위한 고품질의 point clouds를 생성하는 것은 어려운 task이다. 선행 생성 모델들은 Chamfer Distance나 Earth Mover's Distance와 같이 heuristic loss function을 직접 최적화하는 방식으로 접근했고 더 최근 모델들은 GAN, flow 기반의 생성 모델들을 이용하였다. PVD와 같은 논문은 랜덤 노이즈로부터 전체 shape을 생성하는 DDPM방식을 시도하고 있다.

DiT는 이미지 도메인에서 U-Net 기반의 네트워크들을 능가하였고 본 논문은 이를 3D로 확장하고자 한다. 여기에 두가지 어려움이 있으니 (1) Point clouds는 이미지의 pixel과는 다르게 내재적 '순서'가 없다. (2) 3D 공간에서의 추가되는 차원 축은 token 수를 증가시키고 이는 결과적으로 computational cost를 증가시킨다. 각각에 대해 제시하는 해결책은 아래와 같다. 

(1) point clouds는 unordered dataset이다.
기본적으로 image/video는 pixel space에 존재하는 ordered dataset이다. 하지만 point clouds는 그렇지 않다. 이는 선행 연구를 참고하여 voxel representation으로 변형하고 3D positional embedding/3D patch embedding을 고안하였다.

(2) 3D에서 '축'이 하나 추가되면서 연산량이 지나치게 증가한다.
3D window attetntion operator로 vanila global attention 을 대체하여 연산량 폭발 문제를 해결하였다.


### 2. Preliminaries
#### 3D Shape Generation
3D shape Generation은 high-fidelity point clouds 혹은 mesh를 생성하는 것을 목표로 한다. 
* PointFlow:
  two-level의 계층적 분포를 이용하여 CNF로 point  clouds를 생성하고자 하였다.
* ShapeGF:
  Score-matching energy-based network로 points 분포를 학습한다. 초기의 score matching 방식의 모델을 point clouds에 적용한 논문이라고 보면 된다.


* PVD
  PVCNN에 기반하여 DDPM을 적용하였다.
* LION
  point clouds generation을 global shape generation에 detail을 추가하는 point-structured의 계층적 구조로 분리하여 latent space를 diffusion으로 학습한다.



### 3. Method
![fig1](/assets/img/posts/my-paper-review/DiT-3D/DiT-3D_figure2.png)


먼저 문제 정의는 다음과 같이 한다. 각각의 point cloud $\text{p}\_i$에 대하여 전체 point cloud 집합은 $\mathcal{S} = \lbrace{\text{p}\_i}\rbrace^S\_{i=1}$와 같이 나타낸다. $\text{S}$는 전체 데이터의 갯수인데, 논문을 이해할 때는 크게 중요하지 않다. 데이터들은 $M$개의 class label을 갖기 때문에 class label은 $\lbrace y\_i\rbrace^M\_{i=1}$. 와 같이 표현할 수 있다. 각각의 point cloud $\text{p}\_i$는 $x, y, z$좌표값을 갖는 $N$개의 point를 갖는다 $\text{p}\_{i} \in \mathbb{R}^{N \times 3}$. 이 point들은 voxelization을 거쳐 $\text{v}\_i \in \mathbb{R}^{V \times V \times V \times 3}$의 형태로 입력된다. 이렇게 하면 point cloud는 grid한 데이터 형태를 띄게 된다.

이렇게 만들어진 Voxel Data를 Conv3D로 이루어진 patchfication layer를 통과시킨다. 이때 DiT-2D와 마찬가지로, Conv3D의 kernel size에 따라 전체 diffusion 과정에 사용될 feature sequence 길이가 정해진다. 즉, 작은 kernel size를 사용하면 더 작은 지역 단위가 하나의 patch로 묶이게 되고 큰 kernel size를 사용하면 더 넓은 지역 단위로 patch로 묶어 학습하게 된다. DiT-2D와 다른 점은 DiT-2D의 경우 사전학습된 VAE의 latent feature를 patch단위로 나누어 입력 token으로 사용한다. 하지만 3D의 경우 2D와 같은 거대한 데이터셋이 구축되지 않았고 설령 구축되었다 한들 이를 VAE로 학습하는 것은 cost가 매우 큰 일이기 때문에 Voxel단에서 바로 3D patchfy(기본 patch size 4)를 진행한다.

_논문에서 언급하고 있지 않지만, kernel size를 마냥 키우고 줄일수가 없는 것이 기본적으로 nvidia에서 제공하는 cudnn에서 kernel size가 너무 큰 경우는 최적화(Winograd 알고리즘 등)를 해놓지 않았다. 2D CNN을 기준으로 할때, kernel size 1,3일 때가 가장 하드웨어 친화적이고 kernel size 5,7 순서로 최적화가 되어있는 것으로 알고 있다. 이보다 지나치게 큰 경우는 하드웨어 친화적인 최적화가 반영이 안되어 있다. 2D의 경우가 그러하기 때문에 최적화가 덜 되어 있는 3D 연산의 경우 kernel size에 따라 병목현상이 더 크게 걸린다. 또 일반적으로 ViT 계열에서 kernel size는 4로 설정하기 때문에 이를 따르면서 동시에 지나친 병목현상이 생기지 않도록 kernel size 4 patchfy를 진행한 것 같다._


patch단위로 출력된 특징에 순서 정보를 데이터에 추가하기 위해서, 3D positional embedding을 더해준다. 논문에서 제안하는 contribution이긴 하지만 그렇게 특출난 내용은 아니라서 밑의 토글에 내용을 정리했다.

또 CFG를 통해 학습하기 위해서 label embedding정보를 condition으로 timestep 정보 $t$와 함께 추가해준다. 이후는 2D DiT와 마찬가지로 DiT Block을 $\text{N}$개 통과하며 denoising을 학습하게 된다. $\text{N}$개의 DiTBlock을 거친 이후에 마지막 layer에서는 2D-DiT와 마찬가지로 Linear layer를 거친다. 이때 최종 output 형태로 pointcloud를 출력하기 위해서 Devoxelization을 거친다.


#### 3D Window Attention
3D 공간을 학습하기 때문에 증가한 token 갯수는 transformer 방식의 모델의 연산량 측면에서 매우 치명적이다. 따라서 본 논문은 이를 해결하기 위해 3D Window Attention을 제안한다. 
먼저 기본적인 multi-head self-attention 과정을 생각해보자. 

먼저 각각의 헤드 $Q$, $K$, $V$는 동일한 $L \times D$의 차원을 갖는다. 입력 토큰의 길이를 $L = (V/p)^3$라 할때, 아래와 같은 수식으로 표현할 수 있다. 
$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^\top}{\sqrt{D_h}}V\right)$$
$D_h$는 각 head 차원의 크기이다. 계산 복잡도는 $\mathcal{O}(L^2)$를 갖게 되는데, 이는 고해상도 복셀 구조에서 지나치게 부담이 된다. 본 논문에서는 [선행 연구](https://arxiv.org/abs/2203.16527)에서 영감을 받아 2D window attention을 3D로 확장하였다. 이 과정은 전체 입력 토큰의 길이를 줄이기 위해 window size $R$을 사용한다.

쉽게 표현하자면, global attention을 통해 모든 patch token에 대해서 attention을 진행하는 것이 아닌, window size만큼의 국소 부위에서만 attention을 진행하겠다는 의미가 된다. 구체적 수식은 아래와 같다.



$$\hat{K} = \text{Reshape}\left(\frac{L}{R^3}, D \cdot R^3\right) \\
K = \text{Linear}(D \cdot R^3, D)(\hat{K})$$

token들을 윈도우 사이즈 $\text{R}$로 묶고, 이를 Linear layer를 통과시켜 압축한다. 본 논문에서는 선행 연구와 마찬가지로, global attention과 window attention을 혼용하여 global 정보를 손실되지 않게 한다. 중요한 점은, 가중치를 공유하기 때문에 한번에 전역/국소 정보를 모두 모델이 학습하게끔 한다는 것이다. 또한 window attention이 이루어지는 layer들에 한해서 계산 복잡도를 $\mathcal{O}(L^2)$에서 $\mathcal{O}(\frac{L^2}{R^3})$으로 낮출 수 있었다. 논문에서 언급하지 않지만, 추론(생성)시에는 전체 global attention으로 동작하기 때문에 학습에서의 효율성은 챙기지만 추론과정에서의 효율성은 챙기지 못한다.



### 4. Experiment
실험은 선행 연구들과 마찬가지로 ShapeNet에서 chair, Airplane, Car로 비교를 하였다. 각각의 category별 특징은 다음과 같다.
+ Chair:
  가늘고 긴, 혹은 둥근 부분이 공존한다. 대부분은 밑판, 등받이, 4개의 다리가 있는 형태를 떠올리지만 실질적으로 데이터셋을 조사해보면 바퀴달린 의자부터 소파에 더 가까운 형태까지 다양한 형태의 의자가 존재한다. 하지만 오히려 이러한 다양성으로 인해 전체 네모난 복셀에 데이터가 고루 분포한다.
+ Airplane:
  대부분이 여객기의 기체형태를 띄고 있다. 날개가 얼마나 일자인지, 끝이 꺾여 있는지, 얼마나 몸체가 둥글고 길쭉한지 등으로 갈리지만 개중에는 복엽기의 형태를 띄거나 스텔스 폭격기의 형태를 띄는 데이터도 존재한다. 심할 경우 스타워즈에 등장할 것만 같은 형태의 우주선 형태도 있다. 이러한 long-tail 문제가 심각하며 동시에 대부분의 point가 몸체, 날개에 편중되어 있기 때문에 x,y,z 그래프 상에서 variance가 굉장히 작게 나타나는 특징이 있다. 이로 인해 noise를 더하는 스케쥴 자체가 다른 category와 다르게 동작한다.
+ Car:
  네모난 형태, 혹은 살짝 둥글게 한 형태에 바퀴달린 모습으로 모든 데이터가 존재한다. 단 일부 개체는 트럭, f1 형태의 모습을 갖기도 한다. 극단적으로 스케일이 차이나는 데이터가 존재한다. 

평가지표는 선행연구들이 사용하는 지표중 1-NNA, COV 두가지만을 사용했다. 그 이유는 이후 논문들을 읽어보았을 때 해당 두 지표를 제외하면  나쁘게 나왔기 때문이라고 생각된다. 해당 지표들에 대해서는 LION 논문에서 자세히 다루고 있는데, 다른 게시글에서 정리하도록 하겠다.

논문에서 새로 제안하는 3D positional embedding과 3d window attention에 대한 ablation study를 진행했다. 의미 있다고 느끼는 것은 3D window attention이 적용되었을 때 학습 시간이 획기적으로 단축된다는 것이다. 오픈된 코드를 이용하여 직접 학습을 해 보았는데, 해당 보고는 사실인 것으로 확인이 되었다. 하지만 역으로 말해서, 그렇다면 이렇게 큰 모델 구조가 굳이 안필요하지 않을까? 라는 생각을 하게 되었다.

![fig3](/assets/img/posts/my-paper-review/DiT-3D/DiT-3D_table3.png)
2D DiT 구조의 핵심 파트는 그대로 차용했기 때문에 해당 가중치들을 가져다 사용할 수 있었다. 따라서 저자들은 사전학습된 가중치를 이용하여 3D Domain에 학습을 해보았는데, 이때 성능이 유의미하게 유지가 되었다고  보고하고 있다. 도메인이 달라져도 ImageNet 사전학습 가중치가 의미있었던 경우는 이전에도 보고된 바가 있기 때문에 이는 사실인 것으로 보인다. 이때 사용한 Fine Tuning 방식은 DiffFit라는 논문의 방식을 따랐다.(동일하게 화웨이 논문이다)

학습 point cloud의 Category를 하나의 Domain으로 분류하여 각 Category별로 학습한 가중치를 사전학습 가중치로 다른 Category에 FineTuning할 때 효과가 있는지 증명하였다. Airplane은 noise scheduling을 다르게 설정하는데 이 부분에 대한 자세한 언급은 없어서 어떻게 실험을 진행했는지는 확실하게 이해가 되지 않는다.


![fig3](/assets/img/posts/my-paper-review/DiT-3D/DiT-3D_table4.png)
patch size가 작아진다는 것은 더 많은 token을 입력으로 사용한다는 의미이면서 동시에 더 작은 voxel 단위를 diffusion token으로 사용한다는 의미이다. 따라서 성능은 일반적으로 작은 patch size에서 좋은 경향을 보인다. 다만, token 갯수가 세제곱으로 증가하기 때문에 전체 학습/추론 비용이 기하급수적으로 증가하는 문제가 발생한다. 이는 voxel size에서도 같은 경향을 보인다.

model size별로도 비교를 진행하였다. 더 큰 모델일수록 모델의 깊이가 깊어지고, transformer 계열에서 이는 더 좋은 성능으로 귀결된다. 하지만 내가 여러번 S/4를 학습 및 추론해본 결과 해당 모델에서도 시드에 따라 극단적인 성능을 보이며 XL/4보다 좋은 결과를 보이기도 했다. 

### 결론 및 아쉬운 점
논문의 코드를 돌려보면 굉장히 엉망진창 돌아간다. 코드의 구현 자체도 문제이고 시드 고정에 대한 함수도 반영되어있지 않으며, 무엇보다 논문에서 보고하는 성능이 나오지 않는다. 이는 Issue를 둘러봐도 비슷한 문제제기가 꽤 많은 것으로 보아 나의 잘못으로 보이지는 않는다. 코드가 깔끔하지 않다는 것은 곧 논문에서 보고하는 내용에 자신이 없다는 것으로 받아들여진다. 개인적으로 코드를 여러번 돌려본 결과, transformer 모델이 scalibility를 가진다는 것이 수학적으로 이미 증명이 되었음에도 불구하고, 본 논문에서 제안하는 구조에서 scalibility를 확인했다는 것은 신뢰가 가지 않는다. 

또 1-NNA 벤치마크에 대해 오픈리뷰에서는 마냥 작다고 좋은 것이 아니라 50에 가까울 수록 좋으니 이 부분을 수정해달라는 마이너리뷰가 있지만 이를 반영하고 있지 않다. shape generation 논문을 찾아보니 예전에는 50에 가까울 수록 좋았지만 최근 들어 생성모델의 퀄리티가 높아짐에 따라 작으면 작을수록 더 좋은거다라는 암묵적인 합의가 있는 것처럼 보인다. 물론 이러한 암묵적 합의에 동의하지 않는다.

Objaverse를 사용하는 shape generation과 달리 VAE를 사용하고 있지 않다. 현실적인 가장 큰 이유는 학습 시간과 자원의 문제라고 생각이 든다. 

학습 뿐만 아니라 생성에도 많은 시간이 소요된다. 나의 환경에서 S/4를 이용하여 추론을 할 경우 'car'에서 약 40분의 시간이 소요되었다. 이는 생성모델의 효율성 측면에서 해결해야 할 점이라고 생각된다. 

정리하자면, DiT구조를 3D 도메인으로 끌고 온 것은 분명 의미가 있지만, 재현 가능성과 성능 측면에서 그렇게 좋은 논문으로 보이지는 않는다. 하지만 transformer 계열의 모델이기 때문에 추가적으로 적용할 방법들은 많아서 발전 가능성은 높다고 생각된다.
